Design Process

If you would ever ask me, why do I like to be a UX designer I would reply because I want to be creative, for sure, but this is not the only reason. One of the most fashinating thing that I like of UX design is goign through a user-centered design process, being always ready to put my self in the users’ shoes and change my ideas and my designs when necessary. The goal is to create designs for the users, namely for people, and not for my self.

How do I do this?

User centred design process

discovery

  • Conducting competitor analysis
  • Conducting interviews and surveys
  • Conducting user research and analysis

concepting

  • Creating screen-by-screen user flows
  • Coducting card sorting analysis
  • Creating a first draft on the site map

prototyping

  • Creating low-fidelity and mid-fidelity wireframes
  • Develop the most important functions
  • Creating prototypes

testing

  • Testing the prototype/s on potential users
  • Reviewing user data to decide if a new feature is successful
  • Analyzing data and test results for possible solutions
